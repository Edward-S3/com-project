# P4-1: 採点中ローディング表示 実装指示書

| 項目 | 内容 |
|---|---|
| 起票日 | 2026-07-16 |
| 背景 | 面談終了後、審判(採点)完了まで数十秒UIが無反応となり、利用者がスタックと誤認する(2026-07-15 実機確認) |
| 対象 | フロントのみ(app/static/app.js, index.html, style.css)。**バックエンド(main.py等)は変更禁止** |

## 要件

1. 面談終了操作の直後から講評表示(s04遷移)までの間、「採点中」であることを画面に明示する
   - 表示文言: 「採点中です。しばらくお待ちください(最大1分程度)...」
   - スピナー等の動きのある表示を含めること(CSSアニメーションで可)
2. 対象は次の2経路の両方:
   - 経路①(テキスト): end-btn onclick → POST /api/sessions/{id}/end の await 区間(app.js:294付近)
   - 経路②(音声): voice.endSession() 呼出し後、WS "ended" メッセージ受信までの区間(app.js:171付近)
3. ローディング中は end-btn を disabled にする(二重送信防止)
4. 採点失敗時(fetch例外/HTTPエラー)はローディングを解除し、エラーメッセージを表示して end-btn を再度有効化する(リトライ可能にする)
5. 講評表示(renderReport→s04遷移)時にローディングを確実に解除する

## 変更対象ファイル(これ以外の変更は禁止)

- app/static/app.js
- app/static/index.html(ローディング要素の追加)
- app/static/style.css(スピナー等のスタイル)

## 禁止事項

- app/ 配下の Python ファイル、data/、docs/(本指示書除く)への変更
- 既存の画面遷移ロジック(enableNav/showScreen)の仕様変更
- 依存ライブラリの追加(素のJS/CSSで実装)

## 受入基準

1. pytest 46件パス維持(フロントのみの変更なので当然パスすること)
2. テキスト面談で終了→ローディング表示→講評表示、の流れを実機確認
3. 音声面談で同上
4. ローディング中に end-btn が押せないこと
5. git diff が変更対象3ファイルのみであること
