## 1. 現行文(app/static/index.html S-01 同意ブロック)

> 面談ログは保存され総務等が閲覧できます。内容は外部AI APIに送信されます。実在の人事情報・機密情報を入力しないでください。

## 2. 改訂案

> 面談ログ(会話の文字起こし)は社内サーバーにのみ保存され、総務等が閲覧できます。音声そのものは保存されません。
> 面談中の会話内容は、応答生成のため外部AI API(Google等)に送信されます。
> 練習の題材として、ご自身の目標設定の下書きなど実際の業務情報を入力できます。ただし、以下は入力しないでください。
> ・他の従業員の評価・処遇に関する情報
> ・確定済みの人事評価の結果
> ・会社の機密情報(取引先情報・原価・未公開の経営情報等)

## 3. 改訂理由

1. **P4機能との矛盾解消**: 現行文は実在の人事情報の入力を包括的に禁止しており、実際の目標下書きを入力して練習する grounded interview template 構想と両立しない
2. **許容範囲の最小限の緩和**: 入力を許容するのは「本人の・下書き段階の」業務情報に限定。他人の情報・確定済み評価・会社機密は引き続き禁止(禁止対象を3点に具体化し、利用者が判断しやすくした)
3. **事実に基づく記述の正確化**: 保存先が社内サーバー(SQLite: data/sessions.db)のみであること、音声は保存されないこと(トランスクリプトのみ保存)、外部送信は応答生成目的であることを明記

## 4. 事実関係(2026-07-16 確認)

| 項目 | 内容 |
|---|---|
| 保存先 | /opt/interview/data/sessions.db(社内サーバーローカル、SQLite) |
| 音声 | 保存しない。トランスクリプトのみ(SPECIFICATION.md 設計判断5) |
| ログ保護 | OSレベルのアクセス権で保護(非機能要件 N-06) |
| 外部送信 | 面談中の会話内容を応答生成のため Gemini API へ送信(機能上不可避) |
