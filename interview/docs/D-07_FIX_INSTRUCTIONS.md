# D-07修正指示: 審判スコア尺度の整合(judge.py)

## 対象
/opt/interview/app/judge.py のみ。他ファイルの変更は禁止。
変更が必要と判断した場合は着手せず理由を報告し指示を待つこと。

## 背景(証跡確認済み)
- data/rubrics.json の尺度は両role共に 3〜7("scale": [3,4,5,6,7]、
  rating_scale も score 3〜7 の5段階)
- judge.py 16行目 JUDGE_JSON_SCHEMA に「1-7の整数」と矛盾記載
- json.loads(110・117行目)後のスコア範囲検証・クランプが不在

## 修正1: プロンプト尺度表記の動的化
- JUDGE_JSON_SCHEMA の「1-7の整数」というハードコードを廃し、
  rubrics の rating_scale から導出した実際の最小値・最大値
  (現状3〜7)がプロンプトに反映される実装に変更する
- 定数文字列にmin/maxのプレースホルダを設けformatする方式を推奨

## 修正2: スコアのクランプ処理追加
- json.loads 成功後、scores の各値を検証:
  - 範囲外(min未満/max超)は最近傍境界値に丸める
  - 整数でない値は round 後にクランプ
- 丸め・補正が発生した場合、採点結果dictに score_corrections
  フィールドを追加し、[{"perspective": 観点名, "original": 元値,
  "corrected": 補正後値}] 形式で記録する。補正ゼロの場合は
  空リストまたはフィールド省略とする
- リトライ経路(117行目)にも同じ検証を通すこと(処理の共通化推奨)
- 位置づけ: score_corrections は恒常機構ではなく異常検知センサー。
  修正1が機能していれば補正は発生しないはず

## 禁止事項
- data/rubrics.json の変更禁止(人事評価マニュアル準拠のため)
- warnings_json / DB書き込み経路への変更禁止
- judge.py の本修正と無関係な箇所のリファクタリング禁止
