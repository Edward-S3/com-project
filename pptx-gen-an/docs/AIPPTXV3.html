AI駆動型プレゼンテーション資料（PPTX）自動生成アプリケーション仕様書 V3

本書は、ユーザーがアップロードした多様なソースドキュメントをもとに、LLM（Gemini API）を活用して編集可能なPowerPointファイル（.pptx）を自動生成するアプリケーションの仕様書である。

V3では、V2で残存していた「AIが抽象的な指示をサボる（空白やワンパターンを生成する）」という課題を解決するため、Pydanticスキーマによる視覚要素（VisualElement）の厳密な型定義、動的フォントサイズ計算、および生成後の自動品質検証（QA）プロセスを統合した決定版アーキテクチャを採用する。

1. システム概要とアーキテクチャの変更点 (V2 -> V3)

  • 視覚要素の構造化: 曖昧だった「右側のブロック」を廃止し、VisualElementクラスを新設。BIG_STATやSTEP_FLOWなど、描画すべき図解の種類と必要なテキストデータをAPIに強制的に定義させる。
  • 検証駆動生成 (Validator): 生成されたJSONが「同一レイアウトの連続」や「データ不足」を起こしていないかをプログラムで検査し、基準を満たさない場合はGeminiに自動で再生成させる自己修復ループを実装する。
  • 文字はみ出しのプログラム制御: `python-pptx`側で、テキストボックスのサイズと文字数から逆算して最適なフォントサイズを決定するロジックを実装し、物理的なレイアウト崩れを防ぐ。

3. データ構造定義 (src/schemas.py)

V3のコアとなる、視覚要素の種類と必要なデータを分離したスキーマ定義。

from pydantic import BaseModel, Field
from typing import List, Optional, Literal

class SlideTheme(BaseModel):
   palette_name: str = Field(description="パレット名（例: Midnight Executive）")
   dominant_color: str = Field(description="支配色（HEX）")
   support_color: str = Field(description="サポート色（HEX）")
   accent_color: str = Field(description="アクセント色（HEX）")
   is_dark_mode_cover: bool = True

class VisualElement(BaseModel):
   kind: Literal[
       "BIG_STAT",        # 大きな数字 + 説明ラベル
       "STEP_FLOW",       # 番号付きフロー（3〜5ステップ）
       "MATURITY_BAR",    # 水平グラデーションバー
       "ICON_LIST",       # アイコン行リスト
       "COMPARISON"       # Before/After 2段表示
   ] = Field(description="描画する視覚要素の種類")
   
   # BIG_STAT 用
   stat_value: Optional[str] = Field(default=None, description="表示する数値（例: '3.2倍'）")
   stat_label: Optional[str] = Field(default=None, description="数値の説明（例: '3年後の生産性差'）")
   
   # STEP_FLOW / ICON_LIST 用
   items: Optional[List[str]] = Field(default=None, description="各ステップ・行のテキスト（3〜5件）")
   
   # MATURITY_BAR 用
   stages: Optional[List[str]] = Field(default=None, description="段階ラベル（例: ['入門', '中級', '上級']）")
   
   # COMPARISON 用
   before_label: Optional[str] = None
   after_label: Optional[str] = None

class SlideContent(BaseModel):
   slide_number: int
   layout_type: Literal['TITLE', 'TWO_COLUMN_TEXT_AND_BLOCK', 'GRID_2X2', 'GRID_1X3']
   title: str
   bullet_points: List[str] = Field(description="左カラム等の本文テキスト（3〜5行）")
   visual: Optional[VisualElement] = Field(description="右カラムまたは下部の視覚要素（TITLE以外は必須）")

class PresentationData(BaseModel):
   theme: SlideTheme
   slides: List[SlideContent]

5. 新規追加モジュールの仕様

5.1 QA（品質検証）ロジック (src/validator.py)

Geminiが生成したJSONをpptxに変換する前に、以下の検証を行い、エラーがあれば例外を投げてGeminiに再生成（最大2回リトライ）を要求する。

  • 連続レイアウト検知: slides[i].layout_type == slides[i-1].layout_type（TITLE除く）の場合はエラー。
  • 視覚データ欠損検知: visual.kind が要求するデータ（例: BIG_STAT時のstat_value）が None または空文字の場合はエラー。

5.2 フォントサイズ逆算ロジック (src/utils/font_sizer.py)

テキストボックスの物理的な幅と高さから、文字がはみ出さない最大のフォントサイズを動的に計算する。

import math

def calculate_font_size(text: str, box_width_inches: float, box_height_inches: float, base_size: int) -> int:
   MIN_FONT_SIZE = 10
   CHARS_PER_INCH_AT_14PT = 4.0  # 日本語14ptの目安文字数/inch
   
   size = base_size
   while size >= MIN_FONT_SIZE:
       chars_per_inch = CHARS_PER_INCH_AT_14PT * (14 / size)
       chars_per_line = int(box_width_inches * chars_per_inch)
       lines = math.ceil(len(text) / max(chars_per_line, 1))
       height_needed = lines * (size * 1.4) / 72  # 行間の概算高さをインチで算出
       if height_needed <= box_height_inches:
           return size
       size -= 1
   return MIN_FONT_SIZE

6. Streamlit UI 仕様 (app.py)

ユーザー体験を損なわないための最低限の実装要件。

  • 入力: ファイルアップロード（複数可）、スライド生成目的のテキストエリア、スライド枚数スライダー（5〜15枚）。
  • 進行状況: st.spinner を用いて「テキスト抽出中」「Geminiで構成・デザイン生成中（リトライ中）」「スライド描画中」のステータスを可視化。
  • エラーハンドリング: validator.py でのリトライが上限に達した場合、どのスライドで設計上の問題が起きたかを st.warning または st.error でユーザーに明示し、部分的な生成データとともにダウンロード可能にする。
