AI駆動型プレゼンテーション資料（PPTX）自動生成アプリケーション仕様書 V2

本書は、ユーザーがアップロードした多様なソースドキュメントをもとに、LLM（Gemini API）を活用して編集可能なPowerPointファイル（.pptx）を自動生成するアプリケーションの仕様書である。

V2では、AIによる不確実な画像生成（幻覚やデザイン崩れ）への依存を排除し、厳格なプロンプトエンジニアリングと「python-pptx」のネイティブ描画（図形、カラーブロック、タイポグラフィ）を組み合わせることで、ビジネスに耐えうる高品質なデザインを安定して出力するアーキテクチャを採用する。

1. システム概要とアーキテクチャの変更点

本システムは「構成の生成」から「デザインの適用」までをGemini APIとpython-pptxの連携で完結させる。

  • 画像生成AIの廃止: 挿絵などの画像生成は行わず、代わりにカラーブロック、アイコン配置、タイポグラフィの強弱を用いて視覚的なメリハリをつける。
  • 厳格な制約（NEVERリスト）の導入: AI特有の「ダサいデザイン（タイトル下の下線、無意味なカラーバー、単調な白背景の連続）」をシステムプロンプトレベルで完全に封じる。

3. デザインルール定義（システムプロンプトへの組み込み必須要件）

以下のルールをGeminiのsystem_instructionにハードコードし、出力を制御する。

A. カラーパレット制約

  • スライドのカラーパレットは必ず3色以内（支配色60-70%、サポート色、アクセント色）に収める。
  • デフォルトの青、およびクリーム・ベージュ系の背景（#F5F5DC等）は使用禁止。
  • 推奨パレット例: Midnight Executive（#1E2761 / #CADCFC / #FFFFFF）、Charcoal Minimal（#36454F / #F2F2F2 / #212121）などから選択させる。

B. レイアウトとタイポグラフィの制約（サンドイッチ構造）

  • タイトルスライドと最終スライドは「濃色背景＋白文字」、中間コンテンツは「淡色（白）背景＋濃色文字」とする。
  • スライドごとにレイアウトを変化させる（同一レイアウトの連続禁止）。
  • テキストのみのスライドは禁止。必ずカラーブロックやナンバリングコールアウト（巨大な数字）などの視覚的要素を入れる。
  • フォントサイズ: タイトル(36〜44pt / Bold)、見出し(20〜24pt / Bold)、本文(14〜16pt / Regular / 左揃え)。Aptosは使用禁止。

C. 絶対禁止事項（NEVERリスト）

  • タイトル下のアクセントライン（下線装飾）の追加。
  • スライド幅いっぱいのカラーバーやストライプ（ヘッダー・フッター）。
  • 本文テキストの中央揃え。

4. データ構造定義 (src/schemas.py)

デザインルールを反映した新しいPydanticスキーマ。

from pydantic import BaseModel, Field
from typing import List, Optional

class SlideTheme(BaseModel):
   palette_name: str = Field(description="選択したパレット名（例: Midnight Executive）")
   dominant_color: str = Field(description="支配色（HEX形式）")
   support_color: str = Field(description="サポート色（HEX形式）")
   accent_color: str = Field(description="アクセント色（HEX形式）")
   is_dark_mode_cover: bool = Field(description="表紙と最終ページを濃色背景にするか（常にTrueを推奨）")

class SlideContent(BaseModel):
   slide_number: int = Field(description="スライド番号")
   layout_type: str = Field(description="レイアウト ('TITLE', 'TWO_COLUMN_TEXT_AND_BLOCK', 'GRID_2X2', 'BIG_NUMBER_CALLOUT')")
   title: str = Field(description="スライドタイトル")
   bullet_points: List[str] = Field(description="本文テキスト")
   visual_element_text: Optional[str] = Field(description="視覚要素（巨大な数字やキーワード）に入れるテキスト")

class PresentationData(BaseModel):
   theme: SlideTheme
   slides: List[SlideContent]

5. python-pptx 実装時の重要制御ロジック (src/pptx_creator.py)

JSONを受け取り、python-pptxでスライドを生成する際のロジック。

  • ワイドスクリーン設定: prs.slide_width = Inches(13.33) / prs.slide_height = Inches(7.5) を明示する。
  • 余白の絶対確保: 全てのテキストボックスは、スライド端から最低 Inches(0.5) (1.27cm) の余白を設けて配置する。
  • auto_sizeの無効化: テキストのはみ出しを検知・制御するため、text_frame.word_wrap = True とし、文字数から逆算してフォントサイズを固定で割り当てる（AI任せの自動縮小機能は予測不能なレイアウト崩れを起こすため使用しない）。

6. QA（品質検証）プロセス

生成後、以下の検証プロセスをバックグラウンドで実行し、エラーがあれば修正ロジックを走らせるか、ユーザーに警告を出す。

  1. python-pptxのオブジェクトを走査し、テキストボックスの定義高さを超過する文字数が入っていないか計算する。
  2. 可能であれば、生成したpptxをLibreOffice等のCLIを用いてPDF/画像化し、文字のオーバーフローや重なりがないかチェックする機構を設ける。
