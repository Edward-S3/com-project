AI駆動型プレゼンテーション資料（PPTX）自動生成アプリケーション仕様書

本書は、ユーザーがアップロードした多様なソースドキュメントと利用目的（プロンプト）を元に、LLM（Gemini API）および画像生成APIを活用して、編集可能なPowerPointファイル（.pptx）を自動生成するアプリケーションの完全な仕様書である。本仕様書はCursor等のAIエディタに読み込ませて直接開発を開始できるレベルの具体性を備えている。

1. システム概要

本システムは、ユーザーから提供された入力ファイル（PDF、テキスト、Markdown、Word、Excel、PowerPoint）の内容を解析・抽出し、ユーザーが指定したシチュエーションや目的に応じたスライド構成をAIによって自動構築する。さらに、スライドごとのデザインテーマに沿ったアセット画像を生成し、最終的にpython-pptxライブラリを用いて、ユーザーが後から再編集可能なPPTXファイルとして出力するWebアプリケーションである。

4. データ構造定義 (src/schemas.py)

Gemini APIの「Structured Outputs（構造化出力）」機能を用いて、不確定なテキストではなく、プログラムが確実に解釈可能なJSONデータを取得するためのPydanticスキーマである。

from pydantic import BaseModel, Field
from typing import List, Optional

class SlideTheme(BaseModel):
    primary_color: str = Field(description="スライドのメインカラー（HEX形式, 例: '#003366'）")
    secondary_color: str = Field(description="サブカラー（HEX形式, 例: '#4A90E2'）")
    accent_color: str = Field(description="アクセントカラー（HEX形式, 例: '#FF5733'）")
    bg_color: str = Field(description="背景色（HEX形式, 原則として白または極めて薄い色, 例: '#F9F9F9'）")
    font_heading: str = Field(description="見出し用フォント名（例: 'Arial', 'Meiryo'）")
    font_body: str = Field(description="本文用フォント名（例: 'Arial', 'Meiryo'）")
    design_style_keyword: str = Field(description="画像生成AIに渡す共通のスタイルキーワード（例: 'flat vector illustration, corporate minimalist, high quality, corporate blue tone'）")

class SlideContent(BaseModel):
    slide_number: int = Field(description="スライド番号（1から開始）")
    layout_type: str = Field(description="レイアウトの種類 ('TITLE', 'ONE_COLUMN', 'TWO_COLUMN', 'IMAGE_RIGHT', 'IMAGE_LEFT')")
    title: str = Field(description="スライドのタイトル（最大20文字）")
    subtitle: Optional[str] = Field(description="スライドのサブタイトル（任意、表紙や章切り替え用）")
    bullet_points: List[str] = Field(description="箇条書き本文（各項目最大40文字、最大4項目）")
    image_prompt: Optional[str] = Field(description="このスライドに挿入するアセット画像の生成用英語プロンプト。不要な場合はnull。デザインスタイルキーワードは含めず、被写体や構図のみを具体的に記載。")

class PresentationData(BaseModel):
    title: str = Field(description="プレゼンテーション全体のタイトル")
    theme: SlideTheme = Field(description="プレゼンテーション全体で統一するデザインテーマ")
    slides: List[SlideContent] = Field(description="全スライドのコンテンツリスト")

5. アプリケーション処理フロー・アルゴリズム仕様

【Phase 1】UIと複数フォーマットのファイル受付・テキスト抽出 (app.py & src/document_parser.py)

ユーザーからファイルのアップロードを受け付け、拡張子に応じて適切なライブラリを用いて内部テキストに変換する。

●     対応フォーマット拡張: PDF, TXT, MD, DOCX, PPTX, XLSX

●     テキスト抽出ロジックのアルゴリズム:

○     .txt / .md: 標準のUTF-8文字列としてそのまま読み込む。

○     .pdf: pypdf.PdfReaderを用いて全ページのテキストを結合する。

○     .docx: docx.Documentを用いて全段落（paragraphs）および表（tables）のテキストを走査して結合する。

○     .pptx: pptx.Presentationを用いて各スライドのプレースホルダーやテキストボックス内の文字列をすべて抽出・結合する。

○     .xlsx: openpyxl.load_workbookを用いて各シートの有効セル値をCSVライクなテキスト形式（行ごとにカンマ区切り、またはタブ区切り）に変換して結合する。

【Phase 2】構成およびデザイン設計の生成 (src/gemini_client.py)

Phase 1で抽出された全テキスト情報と、ユーザーが画面から入力した「利用目的・ターゲット・状況」のプロンプトを統合し、Gemini APIに引き渡す。構造化出力モード（Structured Outputs）を有効化し、PresentationDataクラスのスキーマに合致するJSONを確実に取得する。

【Phase 3】デザインアセット・画像の生成 (src/gemini_client.py)

得られたJSONデータ内のslidesをループ処理し、image_promptが指定されているスライドに対して画像生成を実行する。デザインの一貫性を確保するため、各スライドのプロンプトの末尾に、共通のテーマキーワードであるtheme.design_style_keywordを自動的に結合してAPI（例: imagen-3.0-generate-002）を呼び出す。生成された画像はoutput/ディレクトリに一時保存される。

【Phase 4】PPTXファイルの組み立てとレイアウト調整 (src/pptx_creator.py)

python-pptxを使用し、白紙のスライドに対して動的に要素を配置していく。

●     文字はみ出し防止アルゴリズム: テキストの文字数（len(pt)）を判定し、30文字を超える場合はフォントサイズを自動的に14ptに縮小、20文字程度であれば16pt、それ以下であれば20ptといったように、テキストボックスの領域内からはみ出さない動的調整ロジックを実装する。

●     カラー適用: 各スライドの背景に対してtheme.bg_color（HEXからRGBに変換）を適用し、タイトルの文字色にはtheme.primary_colorを、強調箇所にはtheme.accent_colorを適用して視覚的な統一感を持たせる。
